## Customisation

- Add city/service-area modifiers ("mole removal seattle", "mole control tacoma") only when harvesting for city-page FAQ gaps — PAA for geo-modified queries is thinner but locally distinctive.
- NEVER seed skin-mole queries. The mole homograph is the #1 disambiguation risk (see `brand_context/target-keywords.md` "Queries to AVOID") — if a harvested PAA question is dermatological ("is my mole cancerous"), discard it at the clustering step, don't answer it.
- New keyword ideas from `related_searches` in harvest output are candidates for this file — add under the right category with a date comment.
