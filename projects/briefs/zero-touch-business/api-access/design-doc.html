Google Ads API Tool — Design Document

Route Ready (routereadykits.com)  •  Internal advertiser tool  •  July 2026

1. Overview

Route Ready sells digital business-template kits for route-based service businesses (cleaning, pressure washing, lawn care) at https://routereadykits.com. This document describes a small internal automation tool that manages our own single Google Ads account via the Google Ads API. The tool is not a product; it is not offered to clients or the public, and no third parties access it or its data.

2. Architecture

The tool is a set of Node.js scripts that run on the owner's workstation, either manually or on a local scheduler (weekly review job). There is no hosted service, no public endpoint, and no multi-tenant component.

  • Auth: OAuth 2.0 installed-application flow. A refresh token is generated once for the manager account login and stored locally in an environment file on the owner's machine. Requests use the developer token from MCC 143-307-0544 with login-customer-id set to the MCC.
  • Transport: Direct REST calls to googleads.googleapis.com (current API version; GAQL via googleAds:search, mutations via the standard :mutate services). No SDK, no third-party wrapper.
  • Scope: All calls target customer 763-085-7815 exclusively.

3. Features

  • Campaign management: create and maintain Search campaigns, ad groups, responsive search ads, and sitelink/callout assets for our kit and guide pages.
  • Keyword hygiene: add/pause keywords; maintain shared negative keyword lists (our niche needs extensive negatives to exclude unrelated queries).
  • Budget guard: read daily spend and enforce a fixed monthly cap (US$75/month) by pausing campaigns if the cap is reached.
  • Reporting: weekly GAQL reports (impressions, clicks, cost, conversions per campaign/keyword) rendered into a local markdown digest for the owner.
  • Conversion upload: upload our own first-party purchase conversions (click-id based) from our checkout provider's sales records.

5. Data handling & compliance

  • Only our own account's advertising data is accessed. No client, public, or third-party access of any kind.
  • Reports are stored locally on the owner's machine; nothing is resold, redistributed, or exposed via any API or UI.
  • No Required Minimum Functionality obligations apply (internal advertiser tool, single account), but standard API terms, rate limits, and policy requirements are followed.
  • Credentials (developer token, OAuth client secret, refresh token) are stored in a local, non-committed environment file readable only by the owner.
