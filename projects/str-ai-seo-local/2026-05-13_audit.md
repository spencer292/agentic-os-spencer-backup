# Got Moles — Full SEO / AEO / GEO Audit

**Date:** 2026-05-13
**Site state:** live since 2026-05-01; trailing-slash canonicalization shipped today (commits `9348db5`, `c0a46f1`); 138 sitemap URLs all 200-in-0-hops.

## Headline verdict

**Got Moles is technically excellent at AI readiness (98/A) and llms.txt (97/A) — and structurally failing at AI citation (44/D), answer extractability (40/D), and brand-fact accuracy (20/F).** The site is built for AI engines to *find* it. It is not yet built for AI engines to *cite* it.

The gap is not technical. The infrastructure layer (schema, bot access, llms.txt, canonicalization) is in the top decile of local-service sites. The gap is downstream: **AI-engine training data is stale and competitor URLs are getting cited instead.** Hallucinations persist because the corrected schema and `llms.txt` (which already carry the right founding year and pricing) have not yet propagated through ChatGPT/Perplexity training cycles. Competitive citations are weak because informational queries are won by WSU/WDFW/Almanac/Iowa State — not by Got Moles' blog.

**Bottom line:** the next 60 days of work shouldn't add more technical schema. It should (a) force AI-engine re-crawl of authoritative pages, (b) close the AEO extractability gap (zero HTML tables, zero ordered lists across 12 sampled pages), (c) build a Services hub page that currently returns 404, and (d) win 3-5 specific informational queries away from WSU.

## Critical issues (launch-critical fixes — max 5)

### 1. `/services/` returns 404 — Services hub missing — **HIGH impact, LOW effort**

- Status: `https://got-moles.com/services/` returns 404 with `meta robots: noindex`
- The three children (`one-time-mole-removal`, `total-mole-control-program`, `commercial-mole-control`) all return 200 with full schema
- **Root cause:** no `page.tsx` at `src/app/(frontend)/services/`
- **Impact:** AEO miss — competitors with a Services hub get topical authority on "mole control services" cluster; AI engines can't extract a service comparison table from a missing page; no internal anchor for "all services" link plan
- **Fix:** Build `/services/` hub page with: BLUF summarizing the three offerings, HTML comparison table (TMCP vs OMP vs Commercial — price, coverage, term, warranty), FAQPage, Service schema with `serviceCatalog`, internal links to each service page. Comparison table doubles as AEO extractability win (currently zero tables sitewide).

### 2. Self-conflicting canonical on `/blog/how-to-get-rid-of-moles/` — **HIGH impact, TRIVIAL effort**

- URL `https://got-moles.com/blog/how-to-get-rid-of-moles/` returns 200
- But its canonical tag points to `https://got-moles.com/how-to-get-rid-of-moles-in-your-yard/` — a different URL
- This is a cornerstone keyword target ("how to get rid of moles" — informational)
- **Impact:** Google deindexes the served URL because the canonical disagrees; rank equity bleeds to a URL that may or may not exist as a legacy redirect
- **Fix:** Update the canonical to match the served URL, OR if the legacy URL is meant to be the canonical, redirect this URL to it and remove from sitemap. Decide which is the source of truth, align all three signals (served URL + canonical + sitemap).

### 3. Zero HTML tables, zero ordered lists across 12 sampled pages — **HIGH impact, MEDIUM effort**

- Pixelmojo AEO recommendation #1 (high priority): "Add HTML tables for comparisons, ordered lists for steps, StatBlocks for statistics"
- Confirmed across home, about, three service pages, three city pages, blog index, three blog posts: **0 `<table>`, 0 `<ol>`** on every single one
- **Impact:** AI engines extract tables verbatim and pull statistics from structured callout blocks. Prose-only content is markedly less citable. This is likely the single biggest reason Got Moles is mentioned but not URL-cited.
- **Fix:** Identify 6-8 pages where a table/list makes intrinsic content sense, not bolt-on. Candidates: Services hub (price comparison), `/about/` (3 services × pricing × coverage), `/blog/best-mole-traps/` (trap comparison), `/blog/types-of-moles-in-washington/` (3-species comparison: Townsend's vs Coast vs Broad-footed), `/services/total-mole-control-program/` (year-round month-by-month schedule), `/faq/` (categorized Q&A table). One ordered-list-of-steps per how-to blog post.

### 4. Schema `dateModified` missing on service + city pages — **MEDIUM impact, LOW effort**

- Schema render check: home + about + blog posts all have `dateModified`
- All three service pages: NO `dateModified`
- All three sampled city pages: NO `dateModified`
- `/blog/` index: NO `dateModified`
- **Impact:** AI engines deprioritize stale content; without `dateModified` the page reads as undateable
- **Fix:** Add `dateModified` to `WebPage` / `Service` / `LocalBusiness` schema on these pages. Use the most recent meaningful content update timestamp from Payload CMS.

### 5. No `Last-Modified` HTTP header on any HTML page — **MEDIUM impact, LOW effort**

- Static `.txt` files (robots.txt, llms.txt, llms-full.txt) have `Last-Modified: Wed, 13 May 2026 07:44:48 GMT`
- Zero HTML pages return `Last-Modified`
- **Impact:** Pixelmojo flags this separately from schema `dateModified`. AI engines treat it as an independent freshness signal.
- **Fix:** Configure Next.js + Vercel to emit `Last-Modified` based on `dateModified` from the schema or Payload CMS `updatedAt`. This is a server header config, not a per-page edit.

### 9. Robots.txt — missing GoogleOther rule

Radar flagged: explicit rule needed for `GoogleOther` (used for Google product research crawls, distinct from Googlebot). Currently it falls through to `User-Agent: *` wildcard.

**Fix:** Add to `robots.txt`:
```
User-Agent: GoogleOther
Allow: /
```
One-line edit.

### 10. Article schema present on blog posts but `BlogPosting` typed — verify Pixelmojo finding

Radar schemaAudit flagged "No Article schema found." Render check shows blog posts emit `BlogPosting` (which IS a child of `Article`). Pixelmojo's check may be strict-matching `"@type": "Article"`.

**Fix:** Either add `Article` as an additional `@type` array (`"@type": ["Article", "BlogPosting"]`) or leave alone — `BlogPosting` is semantically correct. Re-run Radar 7 days later to confirm whether this lifts the AEO score.
