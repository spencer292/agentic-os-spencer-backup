## Brand Disambiguation Status (Step 6) 🚨

**Verdict: PARTIAL — leaky on cost + medical-overlap queries.**

The brand-name "mole" collides with dermatology/skin-mole intent. Search engines and AI assistants split the intent ambiguously:

| Query Type | Status | Evidence |
|---|:-:|---|
| Identity / branded | ✅ clean | "got moles" → animal pest control. Spencer/veteran/Enumclaw entity disambiguates |
| Local + service | ✅ clean | "mole control [city] WA" → pest control, no medical confusion |
| Behavior / biology | ✅ clean | "do moles bite", "what do moles eat" → animal context preserved |
| **Cost / removal** | 🚨 leaky | "how much does mole removal cost" → dermatology dominates. Got Moles invisible despite targeted post |
| **Recurrence / regrowth** | 🚨 leaky | "why do moles keep coming back" → 100% dermatology results. Got Moles invisible |

**Root cause:** Got Moles uses "mole removal" and "moles keep coming back" as cluster anchors, which are precisely the phrases the dermatology industry owns at scale. The pest-intent cluster needs to pivot to "mole CONTROL" / "lawn moles" / "yard moles" / "trapping" phrasing in titles, H1s, and metas. Schema-level disambiguation alone (LocalBusiness Service + serviceArea) hasn't been enough to tip intent.

**Recommended `sameAs` additions** (not all available, but pursue):
- BBB profile URL — needs Spencer to register first
- Wikidata Q-id for "Got Moles" company entity — needs manual creation
- WA Department of Fish & Wildlife mole resource page (cite us) — earned, not paid
- WSU Hortsense reference if we can guest-author — earned, long horizon

## Launch-Critical Fixes (this audit, max 5)

Ordered by impact-per-effort.

### P1.1 — Pivot 5 cost/recurrence cluster posts away from medical-leak phrasing

**Pages affected:** `/blog/mole-removal-cost-washington/`, `/blog/why-moles-keep-coming-back/`, `/blog/diy-vs-professional-mole-control/`, `/blog/best-mole-traps/`, `/blog/monthly-vs-one-time-mole-control/`

**Fix:** Edit titles + H1s + metas + first 100 words to lead with "lawn mole" / "yard mole" / "mole control" / "trapping". Keep "removal" only in long-tail body, never in primary tags.

Example: `How Much Does Mole Removal Cost in Washington State?` → `Lawn Mole Control Cost in Washington (2026 Pricing Guide for Homeowners)`

**Effort:** 1-2 hr per post, 5 posts. ~6 hr total.
**Impact:** Reclaim queries the dermatology industry currently captures at zero effort.

### P1.2 — Add a /blog/repair-mole-damage-lawn/ post

**Why:** Pure content gap. "Mole damage repair" is a high-intent post-removal query Got Moles doesn't address. The Mole Patrol (regional WA competitor) owns it. ~1,500 word evergreen, 3-5 FAQs, mid-flow QuizCTA inherited.

**Effort:** Half-day with `mkt-authority-content` skill.
**Impact:** Captures a query competitors currently own.

### P1.3 — Register BBB profile + add to `sameAs`

**Why:** US service-business entity disambiguation. BBB is a top sameAs signal for AI assistants in the local-services category.

**Effort:** Spencer registers (free or paid accreditation). Then 1-line code change (`src/lib/schema.tsx`).
**Impact:** Disambiguation lift on cost/medical-overlap queries.

### P1.4 — Consolidate 2-hop redirect chains

**Pattern observed:** `/mole-trapping-tacoma/` → 308 → `/mole-trapping-tacoma` → 308 → `/mole-control-tacoma` → 200. Two hops = small equity loss, longer crawl.

**Fix:** Update `redirects.ts` and `next.config.ts` to do a single 301 to the final destination.

**Effort:** 30 min audit + edit.
**Impact:** Marginal but right thing to do — affects ~20+ legacy URLs.

### P1.5 — Verify Yelp 13-review vs Google 219+ delta

**Why:** Yelp Enumclaw shows 13 reviews. Site claims 219+ Google reviews across 3 GBPs. AI assistants triangulate review counts across platforms — discrepancies look like inflated claims to a strict crawler.

**Fix:** Audit each GBP location's actual review count via Google Maps. Add Yelp review pulls to `reviews-testimonials-seo` brief if not already there.

**Effort:** 1 hr (Google Maps verification per GBP) + Spencer.
**Impact:** Trust signal hardening; may reveal review-velocity gap.
