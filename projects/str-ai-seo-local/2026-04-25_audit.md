## Step 2 — GBP Audit Per Location

Per Roy's instruction this pass, Bremerton-as-fourth-location is acknowledged and excluded from launch-blocker list. New build's `BUSINESS` constant (`src/lib/schema.tsx:8`) reflects the canonical Enumclaw HQ; per-city schemas use `cityLocalBusinessSchema` parented to the same Organization.

What's verifiable from code/site:

- **Schema-side:** `aggregateRating: { ratingValue: '5.0', reviewCount: 219 }` hardcoded in BUSINESS constant. **Site-displayed reviews page shows 175** (Seattle 91 + Tacoma 59 + Enumclaw 25). Discrepancy between schema (219) and site UI (175). Both numbers are defensible — 219 is the GBP aggregate, 175 is internal testimonials — but they disagree. Recommend: site copy reads "219+ Google reviews" while reviews page lists 175 internal — clarify in UI which is which, OR sync the schema reviewCount to whichever number is the source of truth for AI assistants.
- **Single phone strategy:** `(253) 750-0211` is the BUSINESS constant phone, applied to root LocalBusiness schema, every city's `cityLocalBusinessSchema`, and on-page CTAs. Acceptable IF underlying GBP records carry distinct verified numbers (call-tracking forwarders fine). Spencer to confirm in GBP dashboard.
- **City schema PostalAddress is incomplete.** `cityLocalBusinessSchema` (line 430) emits only `addressLocality`, no `streetAddress` or `postalCode`. For LocalBusiness rich-result eligibility, full PostalAddress is required. **Not launch-blocking but limits Map Pack rich-result presentation.**

What needs GBP dashboard access (Spencer):
- Per-location: primary category = "Pest Control Service" exactly
- Up to 9 secondary categories per location
- Photos count ≥ 30/location, ≥1 new photo in last 30 days
- Service area as explicit city list, not radius
- Q&A — owner-answered, no orphans
- Veteran-Owned attribute set
- Review velocity tracking (target 8+ new/location/month)

## Launch-Critical Fixes (Max 4)

1. **Wire up the 4 missing schema calls.** `howToSchema` → `/how-it-works/`. `reviewsSchema` → `/reviews/`. `collectionPageSchema + itemListSchema` → `/blog/` and `/service-areas/`. The library code is already written, just not called from pages. **Effort: trivial × 4; effect: +5-8 Authority pillar points.**

2. **Expand `BUSINESS.social` from 3 entries to 8+.** Edit `src/lib/schema.tsx:25-29`. Add Yelp Seattle/Tacoma, Apple Business Connect URL, BBB, Angi, LinkedIn, Google Maps URLs ×3. **Effort: trivial; effect: directly strengthens AI assistant disambiguation; rerendering happens on next deploy.**

3. **Reconcile review count: 219 (schema) vs 175 (UI).**  Pick the source of truth. If 219 is GBP aggregate and 175 is internal testimonials, label them clearly on `/reviews/` ("Internal testimonials: 175. GBP reviews: 219+"). Or unify the number. AI assistants get confused by entity inconsistency. **Effort: trivial copy edit OR moderate if syncing data sources.**

4. **Add streetAddress + postalCode to `cityLocalBusinessSchema`.** Currently emits only addressLocality (line 430). Without full PostalAddress, LocalBusiness rich results don't qualify. Use the canonical Enumclaw address for all city-page LocalBusiness blocks (the GBP that handles each city, with postalCode of the GBP, not the city — this is the standard pattern for SAB businesses). **Effort: moderate (need to map city → GBP first).**
